# 你能管多少个：管理幅度

基础设施搭好了，隔离和集成机制运转正常。下一个问题是：你能同时管多少个 Agent？

管理幅度的上限由验证自动化的成熟度决定。如果每个 Agent 的产出都需要你人工审查，你能管的 Agent 数量受限于你的审查带宽，可能是两三个。如果验证体系足够自动化（第三章的对抗式验证、Trophy 测试、持续集成），你的审查工作缩减为检查差异报告和处理异常情况，管理幅度可以扩大到十个甚至更多。

扩大管理幅度有明确的前置条件。验收标准的覆盖率需要达到一定阈值，否则自动化验证覆盖不到的地方仍然需要人工检查。CI 必须稳定，假阳性率低，否则你的时间花在分辨真假问题上而非解决真实问题。领域需要相对稳定，如果需求变更速率过高，Agent 的 spec 频繁需要更新，你的时间会被消耗在 spec 维护上。任务种类需要有界，如果每个任务都是全新类型的问题，Skill 卡片和已有的 spec 模板无法复用，每次都要从头设计。

当这些前置条件满足时，管理幅度可以通过自动化来扩大。当前置条件不满足时，正确的做法是先投资于前置条件（提高 AC 覆盖率、稳定 CI、标准化任务类型），而非硬着头皮增加 Agent 数量。

在实际操作中，Agent 团队的组成可以以小时为周期调整。这与人类团队以月为周期的调配完全不同。一个 Agent 完成了当前任务，可以立刻被分配到另一个模块。一个模块需要加速，可以临时增加一个 Agent。这种灵活性是 Agent 零边际成本特征带来的直接好处，但有效利用它的前提是你的平台工程和管理机制已经到位。
